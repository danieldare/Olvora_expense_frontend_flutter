# Firebase setup

The app initializes Firebase using **FlutterFire-generated options** from `lib/firebase_options.dart`. No extra config is required if that file and the native config files are present and match your app.

## If you see a Firebase initialization error

1. **Install FlutterFire CLI** (if needed):
   ```bash
   dart pub global activate flutterfire_cli
   ```

2. **Configure Firebase for this project**:
   ```bash
   flutterfire configure
   ```
   This will:
   - Link or create a Firebase project
   - Register your app (Android package name and iOS bundle ID)
   - Generate/update `lib/firebase_options.dart`, `android/app/google-services.json`, and `ios/Runner/GoogleService-Info.plist`

3. **Bundle ID / package name must match**:
   - **Android**: The package name in Firebase Console must match `applicationId` in `android/app/build.gradle.kts` (e.g. `com.example.frontend_flutter_main` or your production ID).
   - **iOS**: The bundle ID in Firebase Console must match `PRODUCT_BUNDLE_IDENTIFIER` in Xcode (e.g. `com.olvora.expense`). If your app uses a different bundle ID than the one in the current `firebase_options.dart`, add that app in [Firebase Console](https://console.firebase.google.com) under your project, then run `flutterfire configure` again so it regenerates the config files for that app.

4. **After adding or changing an app** in Firebase Console, run `flutterfire configure` again so `firebase_options.dart` and the native config files stay in sync.

## Files involved

| File | Purpose |
|------|--------|
| `lib/firebase_options.dart` | Generated by FlutterFire CLI; used at runtime for `Firebase.initializeApp(options: ...)`. |
| `android/app/google-services.json` | Android config (downloaded/updated by `flutterfire configure`). |
| `ios/Runner/GoogleService-Info.plist` | iOS config (downloaded/updated by `flutterfire configure`). |

Do not edit `firebase_options.dart` by hand; regenerate it with `flutterfire configure`.
